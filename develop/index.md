These docs are for **contributors** — people who want to read, modify, or extend the [flowlog-compiler](https://github.com/flowlog-rs/flowlog/releases) and [flowlog-build](https://crates.io/crates/flowlog-build). If you just want to *use* <StyledFlowLog />, the [Tutorial](/tutorial/intro) is what you want instead.

## Prereqs

You'll get the most out of these docs if you:

- can read Rust comfortably (you don't need to be an expert; you need to be able to follow a function);
- have passing familiarity with Datalog — what a relation is, what a rule is, what a fixpoint is;

You do **not** need prior knowledge of Differential Dataflow, Timely Dataflow, or compiler engineering. Those are introduced where they first matter.

If you're new to Datalog, start with the [Tutorial](/tutorial/intro) before coming back here — it's the shortest path to the vocabulary you'll need.
